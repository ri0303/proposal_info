# proposal_info — Claude Code エージェント設定

## 実行手順

### 1. 日付・曜日確認
```bash
date +%Y-%m-%d   # 今日の日付
date +%u         # 曜日番号（1=月 〜 7=日）
```

### 2. Web 検索（曜日に応じたエリア）

#### 2-0. 集約サイトを最初に確認（最優先・必須）★★
個別自治体の検索より先に、**公募中案件をまとめている専門サイト**を確認する。これらは締切日が明記されており、終了済み案件が混入しにくい。

| サイト | 確認方法 |
|--------|---------|
| `architecturehack.com` | 対象エリアの案件を検索、締切日・参加表明期限が明記 |
| `kenchiku.co.jp/compe` | 建築コンペ・プロポーザル一覧（JIA系） |
| `propopers.studio-qtn.net` | 公募速報ログ、エリア・種別で絞り込み |

検索クエリ例:
- `site:architecturehack.com [都道府県名] プロポーザル`
- `kenchiku.co.jp compe [都道府県名] 2026`

#### 2-1. 建設専門紙で新規公告を確認
「いま公募が始まった」案件は**建設専門紙の記事**として掲載される。結果報告より公告ニュースを優先して検索する。

- `kensetsunews.com [都道府県名] プロポーザル 公募 2026年[月]`
- `decn.co.jp [都道府県名] 建築設計 プロポーザル 令和8年`
- `nikoukei.co.jp [都道府県名] プロポーザル 公告 2026`

#### 2-2. 自治体サイト検索（ネガティブキーワード付き）
**平日（月〜金）**
- `site:*.lg.jp [都道府県名] 建築設計 プロポーザル 令和8年 -選定結果 -審査結果 -特定`
- `site:*.lg.jp [都道府県名] 設計者選定 公募 -決定 -契約候補者`
- `[都道府県名] 市町村 建築設計 プロポーザル 公募 令和8年 2026年`
- `[都道府県名] 庁舎 学校 設計競技 プロポーザル 2026 -選定結果`

**土日（民間）**
- `建築 設計コンペ 民間 公募 2026 -結果`
- `建築設計 プロポーザル 民間 募集 令和8年`
- `JIA 建築賞 コンペ 公募 2026`
- `新建築 住宅コンペ 設計競技 2026`

### 3. フィルタリング
- **対象**: 官公庁施設・文化施設・教育施設・庁舎・集合住宅 / 民間コンペ
- **除外**: 提案締切が本日より前の案件 / 小規模リフォーム・内装のみ

### 3.5 裏取り検証（必須）★
Web検索結果のスニペットには「年」が省略され「○月○日締切」のみ表示される場合が多い。これを当年（本日基準）と決め打ちしないこと。「案件一覧」（募集中）に載せる前に、各案件で以下を行う:

1. **終了済みチェックを先に行う（最重要）**: 候補案件が見つかったら、まず `[案件名] 選定結果` / `[案件名] 審査結果` / `[案件名] 契約候補者` / `[案件名] 特定` で追加検索する。これらがヒットする＝すでに終了済みのため、「案件一覧」ではなく「※参考：締切済み案件」に振り分ける。**終了チェック前に案件一覧へ記載しない。**
2. **年の確定**: 「公告日」「令和○年」「20XX年」が明記された情報を確認し、締切日の年を確定する。年が確定できない場合は「案件一覧」には載せず、「要確認」として扱う（注目案件や所見での参考紹介は可）。
3. **検索クエリの年指定**: 検索クエリには「令和8年」「2026年」など今年度を明示し、古い年度（令和6年・2024年等）の結果が混在していないか確認する。
4. **ページ直接確認**: WebFetch でページが取得できた場合は「募集中」「参加受付」「提案書受付期間」などの文言を確認し、終了していないかをチェックする。403で取得できない場合でも、スニペットに「選定結果」「最優秀提案者」「決定しました」などの文言がないか確認すること。

### 5. HTML レポート自動生成 ★
Markdown 作成後、必ずスクリプトで HTML を生成する:
```bash
python3 scripts/md_to_html.py reports/YYYY-MM-DD.md
```
→ `reports/YYYY-MM-DD.html` が自動生成される。

### 6. コミット・push
```bash
git config user.email "03ryoisaka03@gmail.com"
git config user.name "proposal-scout-bot"
git add reports/YYYY-MM-DD.md reports/YYYY-MM-DD.html
git commit -m "daily proposal report: YYYY-MM-DD [エリア名]"
git push -u origin main
```
git push が HTTP 503 で失敗した場合は、`mcp__github__push_files` ツールを使って両ファイルを push する。

## HTML 変換スクリプトについて
`scripts/md_to_html.py` は Python 3 stdlib のみで動作する。

- Markdown の見出し・テーブル・箇条書き・インラインリンク・太字を HTML に変換
- 締切日カラムをステータスバッジ（黄=要確認 / 赤=締切済み / 緑=期限内）に自動変換
- `## サマリー` の収集件数から概要カードを自動生成
- `## 注目案件` の `### N. タイトル` + `- **ラベル**: 値` 形式をカード UI に変換

テンプレートの構造（セクション名）を変更するとパースが崩れるため注意。

## 注意事項

- 取得できなかった都道府県がある場合は `## エラー・注意事項` に記載
